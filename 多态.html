多态（Polymorphism）是面向对象编程中的一个重要概念，它允许同一个类型的对象在不同情境下表现出不同的行为。多态性使得程序更灵活、可扩展，并提高了代码的可维护性。
有两种主要类型的多态性：
编译时多态性（Compile-time Polymorphism）：也称为静态多态性或方法重载。在编译时，编译器根据方法的签名（方法名、参数类型、个数）来确定调用哪个方法。这种多态性发生在编译阶段，因此也称为早期绑定。
示例代码（方法重载）：
java
public class Example { public void printMessage(String message) { System.out.println("Message: " + message); } public void printMessage(int number) { System.out.println("Number: " + number); } }
运行时多态性（Runtime Polymorphism）：也称为动态多态性或方法重写。在运行时，通过对象的实际类型来确定调用哪个方法。这种多态性发生在运行时，因此也称为晚期绑定。
示例代码（方法重写）：
java
class Animal { public void sound() { System.out.println("Animal makes a sound"); } } class Dog extends Animal { public void sound() { System.out.println("Dog barks"); } } class Cat extends Animal { public void sound() { System.out.println("Cat meows"); } }
在运行时，通过以下方式实现多态性：
java
Animal dog = new Dog(); Animal cat = new Cat(); dog.sound(); // 调用Dog类的sound方法 cat.sound(); // 调用Cat类的sound方法
多态性提供了一种灵活的设计方式，允许你编写更通用、可扩展的代码。
